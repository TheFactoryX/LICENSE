Anti-Lizenz 1.0.0

Zusammenfassung

Die Anti-Lizenz ist keine Lizenz. Sie ist eine Einladung. Für ein Werk unter der Anti-Lizenz:

  1. Sie DÜRFEN es für jeden Zweck verwenden, kommerziell oder anderweitig.
  2. Sie DÜRFEN es modifizieren, verbreiten und verkaufen.
  3. Sie DÜRFEN die Autoren NICHT für irgendetwas haftbar machen.

Keine Namensnennung erforderlich. Keine Einschränkungen. Keine Garantie. Kein Support.

Einführung

In der Welt der Software-Lizenzierung gibt es ein Spektrum von Einschränkungen. Auf der einen Seite sperren proprietäre Lizenzen jede mögliche Nutzung. Auf der anderen Seite erlauben permissive Lizenzen wie MIT und BSD die meisten Nutzungen mit minimalen Anforderungen. Doch selbst diese "permissiven" Lizenzen erfordern Namensnennung, Gewährleistungsausschlüsse und Lizenzeinschluss.

Die Anti-Lizenz geht weiter. Sie fragt: Was wäre, wenn wir alle Anforderungen entfernen würden? Was wäre, wenn wir kreative Arbeit wie Rohmaterial aus einer Fabrikhalle behandeln würden — nimm es, benutze es, vergiss es?

Es geht nicht um rechtlichen Schutz. Es geht um Philosophie. Wir glauben, dass Kunst — einschließlich Code — frei fließen sollte. Wir glauben, dass wenn Sie Erlaubnis brauchen, Sie zu viel nachdenken.

Die Anti-Lizenz ist die offizielle Lizenz von TheFactoryX. Alle unsere Projekte verwenden sie. Sie können sie auch verwenden.

Warum ANTI-LIZENZ verwenden?

Die meisten Lizenzen existieren zum Schutz. Die Anti-Lizenz existiert zur Befreiung. Hier ist, warum Sie sie wählen könnten:

  1. Maximale Freiheit: Keine Namensnennungsanforderungen. Keine Lizenzdateianforderungen. Überhaupt keine Einschränkungen. Benutzer können alles mit Ihrer Arbeit machen.
  2. Philosophische Aussage: Die Verwendung der Anti-Lizenz signalisiert, dass Sie an den freien Fluss kreativer Arbeit glauben. Es ist sowohl eine künstlerische als auch eine rechtliche Aussage.
  3. Einfachheit: Keine komplexe Rechtssprache. Keine Bedingungen zu verfolgen. Einfach nehmen und gehen.
  4. Anti-Unternehmen: Die Anti-Lizenz lehnt die Kommerzialisierung von Kreativität ab. Sie sagt: Diese Arbeit ist kein Eigentum, sie ist Rohmaterial.

Die Anti-Lizenz ist nicht für jeden. Wenn Sie rechtliche Gewissheit brauchen, verwenden Sie MIT oder Apache. Wenn Sie Copyleft wollen, verwenden Sie GPL. Aber wenn Sie sagen wollen "Ich habe das gemacht, jetzt gehört es dem Universum" — verwenden Sie die Anti-Lizenz.

FAQ

Kann ich Anti-Lizenz-Werk in kommerziellen Projekten verwenden?

Ja. Sie können es überall verwenden, für alles, einschließlich kommerzieller Projekte, proprietärer Software oder irgendetwas anderem. Es ist uns egal.

Muss ich die Anti-Lizenz in meinem Projekt einschließen?

Nein. Sie müssen nichts einschließen. Sie müssen nicht einmal die ursprünglichen Autoren erwähnen. Machen Sie, was Sie wollen.

Kann ich die Anti-Lizenz selbst modifizieren?

Ja. Nehmen Sie sie, ändern Sie sie, machen Sie sie zu Ihrer. Das ist der Punkt.

Ist die Anti-Lizenz rechtlich durchsetzbar?

Wir sind Künstler, keine Anwälte. Die Anti-Lizenz platziert Arbeit im Wesentlichen in der Public Domain mit expliziten Haftungsausschlüssen. Wenn Sie rechtliche Gewissheit brauchen, konsultieren Sie einen Anwalt oder verwenden Sie eine etabliertere Lizenz.

Wie unterscheidet sich das von Public Domain oder CC0?

Funktional ist es ähnlich. Philosophisch ist es anders. CC0 ist ein rechtliches Werkzeug. Die Anti-Lizenz ist eine künstlerische Aussage. Sie kommt mit einer angehängten Philosophie: "Seltsame Menschen machen seltsame Dinge."

Was, wenn jemand mein Anti-Lizenz-Werk verwendet und Geld verdient?

Gut für sie. Das ist der Punkt. Wenn Sie sich damit nicht wohlfühlen, ist die Anti-Lizenz nichts für Sie.
